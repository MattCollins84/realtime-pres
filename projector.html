What is "The Real-time Web"?

The real-time web is a network web using technologies and practices that enable users to receive information as soon as it is published by its authors, rather than requiring that they or their software check a source periodically for updates.

What does this mean?

  • "Push" as well as "Pull"
  • Event driven

What are Websockets?

  • Persistent tcp connection between client and server
  • Full Duplex (two way) communication
  • Stateful - bad for horizontal scaling
  • Very lean, each request framed by 2 bytes
  • Originally part of the HTML5 effort, but now has a standalone specification
  • Accessible via Javascript

Compared to HTTP

  • Temporary connection, until request is dealt with
  • Uni-directional, pre-determined pattern
  • Client makes a request, server sends a response
  • Stateless - good for horizontal scaling
  • Quite verbose, each request framed by 500+ bytes

What does this actually mean?

  • No more polling!
  • Drastically reduces unnecessary network traffic
  • As well as reducing DB/Server hits
  • Scales amazingly well
